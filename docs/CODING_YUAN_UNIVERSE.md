## One-Line Positioning

Coding猿 Universe is an AI Worker control plane where real agents appear as memorable operator characters, do real work, expose their state, and leave auditable traces.

It is not an NFT monkey skin. It is not a chatbot wrapper. It is a visual operating system for AI labor.

Consumer translation:

> A 24/7 AI worker company you can see, manage, trust, and show.

## Core Thesis

Most AI tools still feel like text boxes. Users see prompts, terminals, and logs, but they do not feel that a worker is alive, busy, blocked, or accountable.

Coding猿 turns agent work into a visible workplace:

- Work becomes spatial.
- Agent state becomes legible.
- Reliability becomes observable.
- The product earns memory through character.

The IP is the doorway. The moat is the runtime, task evidence, orchestration, and trust layer behind the doorway.

## IP Rules

Do:

- Make every worker role distinct by job, tools, motion, and dashboard behavior.
- Make role identity explain function: appearance, posture, personality, workspace, and motion should tell the user what kind of work is happening.
- Keep the tone premium, tactical, technical, and slightly underground.
- Use the ape metaphor for focus, stamina, engineering grit, and late-night execution.
- Show real work artifacts: tests, diffs, alerts, PRs, traces, revenue, risk.

Do Not:

- Copy BAYC traits, names, logos, outfits, or exact visual language.
- Make it look like a meme NFT gallery.
- Let the mascot replace product substance.
- Use cute toy proportions as the primary product style.

## Product Pillars

1. Visible Labor

Every agent has a body, posture, state, and work surface. Users understand the state before reading logs.

2. Real Runtime

Workers must map to actual tools: Codex, Claude, MCP servers, browser automation, terminal commands, CI, GitHub, Datadog, internal APIs, and local desktop actions.

3. Auditability

Every autonomous action produces evidence: inputs, tools used, files touched, tests run, risk decisions, and final output.

4. Trust and Control

Users can pause, inspect, approve, replay, and constrain agents. High-risk actions require explicit approval.

5. IP Compounding

The same worker identities should appear in product UI, screenshots, short videos, docs, launch pages, and share cards.

6. Real Work Over Theater

The product must optimize for two truths at once: users need to feel the workers are alive, and the workers must actually improve throughput, reliability, or revenue. If a visual element does not map to state, evidence, risk, or output, it is decorative debt.

## Product Shape

Stage 1: AI Worker Desktop

- User opens one dashboard.
- Sees Coding猿 and other workers in a live workspace.
- Can inspect projects, tasks, logs, risks, and outputs.
- Current `codex-office` is the prototype base.

Stage 2: AI Worker Company

- Add worker roles, departments, handoffs, and manager views.
- Tasks move between workers.
- Each worker has skills, tools, reliability score, and memory.

Stage 3: AI Operating System

- Workers execute real jobs across local machine, cloud, GitHub, monitoring, finance, and browsers.
- The control plane becomes the user's AI company.
