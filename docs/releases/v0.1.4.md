# v0.1.4 - Public Feedback Template

This release adds a read-only public feedback template command for external pilot testers.

The goal is to reduce the last step from "I ran the pilot" to "I can safely post a redacted result." The command does not record a tester result, inspect project files, call an AI model, or write files.

## What Changed Since v0.1.3

- Added `npm run pilot:feedback-template`.
- Added `scripts/pilot-feedback-template.mjs`.
- Updated README, `docs/pilot/START_HERE.md`, and `docs/pilot/TESTER_INTAKE_CHECKLIST.md`.
- Included v0.1.4 notes in the OSS application packet source.
- Added test coverage for the safe template output.

## How To Use

```bash
git clone https://github.com/guamee16888/codingape-office.git
cd codingape-office
npm install
npm run pilot:smoke
npm run dev
```

After the first run:

```bash
npm run pilot:feedback-template
```

Paste the generated template into:

```text
https://github.com/guamee16888/codingape-office/issues/5
```

## Safety Notes

The feedback template command is intentionally read-only. It:

- does not write files
- does not read project files
- does not call model providers
- does not record tester results
- tells testers not to post API keys, `.env` contents, private keys, wallet files, certificates, private paths, or private source
