# v0.1.2 - Self-Service Pilot Start

This release makes the external pilot easier to try without waiting for a private invite.

Codingape Office is still an early local-first AI coding worker for macOS. The safety promise is unchanged:

- selected project folder only
- evidence before patch
- diff before write
- verification before apply
- Human Gate and Apply Gate before project writes
- rollback/report available

## What Changed Since v0.1.1

- Added a short first-run pilot entry point: `docs/pilot/START_HERE.md`.
- Added a `10-Minute Pilot` section near the top of the README.
- Linked the public pilot tracker from the first-run path.
- Kept feedback instructions focused on redacted, safe tester reports.

## Start Here

External testers should begin with:

```text
docs/pilot/START_HERE.md
```

The first run should use a toy repo, test repo, or disposable branch.

Useful feedback is either:

- a visible diff with Human Gate understood, or
- a clear blocker category with redacted feedback.

## Safety Notes

Do not post API keys, `.env` contents, private paths, private source, support bundle raw contents, screenshots with secrets, or personal contact details in public feedback.

Codingape Office still does not:

- automatically apply code changes
- scan the full disk by default
- upload whole projects by default
- bypass Human Gate or Apply Gate
