# v0.1.3 - Pilot Smoke Check

This release adds a lightweight source checkout smoke check for external testers.

The goal is to help a new tester confirm their local setup before starting the 10-minute pilot. The command does not start the app, does not select a project, does not call an AI model, and does not write to user project files.

## What Changed Since v0.1.2

- Added `npm run pilot:smoke`.
- Added `scripts/pilot-smoke-check.mjs`.
- Updated `docs/pilot/START_HERE.md` to run the smoke check before `npm run dev`.
- Updated the README quick-start path to include the smoke check.
- Added test coverage that the smoke check runs without exposing local paths or API-key-like strings.

## How To Use


## Safety Notes

The smoke check is intentionally read-only. It checks:

- Node.js version
- npm availability
- git availability
- source checkout shape
- key pilot files
- key npm scripts
- whether the default local port appears available

It does not inspect browser sessions, cookies, API keys, `.env` files, private source, or support bundles.
