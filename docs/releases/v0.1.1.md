# v0.1.1 - Governance And CI Signal Release

This release documents the first public governance hardening pass for Codingape Office.

Codingape Office remains an early local-first AI coding worker for macOS. The product promise is unchanged:

- choose a project folder
- collect evidence
- generate a plan and diff
- run verification
- require Human Gate and Apply Gate
- keep rollback available

## What Changed Since v0.1.0

- Added public issue and pull request templates.
- Added `CONTRIBUTING.md` and `SECURITY.md`.
- Added an OpenAI OSS application packet generator.
- Added external pilot tracking for first-run feedback.
- Added a safe pilot tester result recorder.
- Added PR-only maintainer governance documentation.
- Added GitHub Actions CI with `npm test`.
- Enabled solo-maintainer governance for `main`:
  - pull requests required
  - `npm test` required
  - strict status checks enabled
  - admin enforcement enabled
  - force pushes disabled
  - branch deletion disabled

## Current Truthful Status

- Real external tester results: 0 recorded.
- OpenAI OSS application submission: pending maintainer action.
- First 3 tester invites: pending real messages.
- App Store distribution: not complete.
- Mac app packaging exists, but this release is source-focused.

## Safety Notes

Codingape Office still does not:

- automatically apply code changes
- scan the full disk by default
- upload whole projects by default
- bypass Human Gate or Apply Gate
- include API keys or private local paths in public tracker data
